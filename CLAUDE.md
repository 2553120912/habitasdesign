# HB 建筑事务所网站 — Claude 工作指引

## 项目概述

为一家小型建筑事务所构建简洁高级的展示型官方网站。用户是不懂代码的小白，需要 Claude 以清晰、耐心、稳扎稳打的方式协作。

## 工作铁律

1. **不要一口气做太多** — 每次改动控制在 1-3 个文件，改动范围事先确认
2. **经常停下来让用户检查** — 每完成一个可感知的变化，打开浏览器让用户看效果
3. **先规划再动手** — 任何非微小改动必须先写计划、获得批准再实施
4. **不改超出范围的** — 用户说改 A，就只改 A，不"顺便优化"B
5. **说人话** — 用户不懂代码，用普通人能理解的语言解释做了什么

## 开发日志

每次开发结束后，在 `devlog/` 目录下创建或更新当日日志（文件名：`YYYY-MM-DD.md`）。

日志格式遵循 `docs/workflow.md` 中的规范。

## 工作流程

```
用户提需求 → 读相关 docs → 写计划 → 用户确认 → 实施 → 打开浏览器检查 → 更新日志
```

计划文件统一放在 `C:\Users\A\.claude\plans\` 下，通过 Plan Mode 创建和批准。
