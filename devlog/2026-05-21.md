### 阶段十 — 模块滚动渐入（含 bug 与修复）

- [x] 需求：每个内容模块**首次**进入视口时，文字渐入 2 秒（一次浏览内只触发一次）
- [x] 实施方案：
  - CSS `.scroll-reveal { opacity: 0; transition: opacity 2s ease-out }` + `.is-revealed { opacity: 1 }`
  - 5 个内容模块加 `scroll-reveal` class
  - JS 用 `IntersectionObserver`（threshold 0.15）监听，进入视口加 `is-revealed` + `unobserve` 一次性触发
- [x] **首版 bug**：用户反馈「向下滑时 Hero 文字反复出现，别的模块文字没有渐入效果」
  - **根因诊断**：Hero 是 `position: sticky; z-index: 1; 黑底` 一直贴顶部；5 个内容模块 `z-index: 2` 在它之上。当模块 `opacity: 0` 时整块透明，用户透过透明模块看到底下 sticky 的 Hero 黑底白字 → 视觉上「Hero 文字反复出现」。同时用户注意力被前景 Hero 文字夺走，反而没意识到背景模块正在淡入
  - **修复**：CSS 选择器从整块 `.scroll-reveal` 下移到内部文字元素 `.scroll-reveal :where(h1, h2, h3, h4, p, span)`。模块背景立即显示（不再透视）、图片立即显示，**只有文字** 2 秒淡入——这正契合用户原话「字也渐入」
- [x] **已知小瑕疵**：故事板块「我们的故事 →」是 `<a>` 直接包文本节点，没 `<span>` 包裹，不在选择器内 → 不渐入。用户验收时表示「可以了」，暂不处理

## 备注

- **用户工作风格反复观察确认**：要求每个改动 1-3 个文件、改完打开浏览器看效果、Plan Mode 走完整流程。今日累计批准 10+ 个 plan
- **「opacity 整块淡入 × sticky 元素 = 透视 bug」教训**：sticky 元素（如本站 Hero）一直贴在屏幕顶部，如果上层模块用 `opacity: 0 → 1` 整块淡入，透明期间会**透出** sticky 元素的视觉内容，产生「鬼影」效果。修复套路：opacity 加在文字/前景元素上，背景容器保持不透明
- **方向选择 vs 工程质量**：阶段九的边缘羽化方案技术实现是干净的（mask-image + rAF + IntersectionObserver-friendly），但用户审美方向不喜欢就该果断撤回。工程对，不等于方向对——别因为「代码写得不错」就死磕
- **Stacking Context 教训**：CSS `z-index` 不是全局属性。父元素一旦创建 stacking context（如 `position: fixed + z-index 非 auto`），子元素的 z-index 就被关在这个盒子里。要真正浮在全局最上层，要么提父级 z-index，要么把元素移出该祖先
- **几何修复 vs 美学修复**：今日两次「形状不对」都是几何精度问题（X 错位 1px、中英按钮高低差），不是审美问题。先核算数值再调
- **作品集呈现思路明朗化**：从 3 列网格 → 一排一个大图，体现 SANAA 风格的「每个作品都是独立舞台」。剩余两个项目按 Lovell House 模式逐个填充即可
- **第一张真实图上线节点**：今天网站从「纯占位」迈到「有真实作品」第一步，封面渲染图（白色立方体 + 黑窗框 + 围墙绿植）质感与极简黑白灰站点风格契合
